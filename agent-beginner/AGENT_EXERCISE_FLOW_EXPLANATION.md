# Agent-Beginner 两个练习到底在做什么

这份说明的目标只有一个：把练习一和练习二分别画成最简单的流程图，让它们和“电脑操作 Agent”区分开。

## 先说结论

这两个练习都不是在让你做 `OpenClaw` 那种“控制电脑、点按钮、操作桌面”的 Agent。

它们更接近下面两类：

1. 练习一：业务型工具 Agent
2. 练习二：研究型 / RAG 型 Agent

换句话说，这份作业里的“智能体”更像：

```text
大模型 + 你定义的工具 + 文档输入 + 一点状态/数据库
```

而不是：

```text
大模型 + 鼠标键盘控制 + 屏幕截图 + GUI 自动化
```

---

## 练习一：智能体体验与原理初探

### 这题本质上在做什么

这题是在做一个“个人日程助手”。

关键不是让模型随便聊天，而是让它：

1. 看懂用户要做什么
2. 调用你自己写的工具
3. 把日程真正创建、修改、查询、删除
4. 能从聊天记录、飞书文档、Markdown 笔记里抽取日程

### 最简单的流程图

```text
用户输入一句话 / 给一个飞书文档 / 给一份 Markdown
                |
                v
          大模型判断任务
                |
                v
      选择调用哪个日程工具
                |
                v
  create / update / query / delete / ingest
                |
                v
       本地 SQLite 或飞书日历落地
                |
                v
          把结果返回给用户
```

### 如果细一点，可以理解成这样

```text
聊天记录 / 飞书文档 / Markdown
            |
            v
      抽取出时间和标题
            |
            v
  解析成结构化事件（开始时间、结束时间、说明）
            |
            v
     调用日程工具写入或更新
            |
            v
   返回“创建成功 / 更新成功 / 冲突”
```

### 所以这题不是在做什么

不是：

```text
AI 打开飞书窗口
-> 自己移动鼠标
-> 点日历按钮
-> 像人一样操作界面
```

而是：

```text
AI 读懂文字
-> 调用你写好的程序接口
-> 完成日程处理
```

### 一句话理解

练习一是：

```text
“让大模型学会调用你写的日程工具”
```

## 练习二：NexDR 的理解与修改

### 这题本质上在做什么

这题不是让你重新造一个 Agent，而是让你在已有 NexDR 研究系统上做三类改造：

1. 改搜索
2. 改输出形式
3. 改输入来源

它更像一个“研究报告生成流水线”。

### NexDR 原始流程可以理解成

```text
用户提出研究问题
        |
        v
   系统去搜索资料
        |
        v
   整理内容并生成报告
        |
        v
 输出 markdown / html
```

### 题目要求你改成

```text
用户提出研究问题
        |
        v
 改成用 Semantic Scholar 搜论文
        |
        v
 读取更多输入：txt / md / pdf / image
        |
        v
 只生成 markdown，不生成 html
        |
        v
 用户手动修改 markdown
        |
        v
 Agent 识别修改点，再继续修订
```

### 最简单的流程图

```text
问题 + 输入文件
     |
     v
 搜索论文 / 读取文档
     |
     v
 整理证据并生成 markdown v1
     |
     v
 用户修改 markdown v1
     |
     v
 Agent 比较差异
     |
     v
 输出 markdown v2
```

### 它和普通 RAG 很像

你可以把练习二理解为：

```text
一个“研究型 RAG Agent”的裁剪和改造
```

只是这里不只是“检索然后回答”，而是：

1. 要能生成一份报告
2. 报告能被用户编辑
3. Agent 要理解编辑后的差异继续工作

### 一句话理解

练习二是：

```text
“修改一个研究报告 Agent 的流水线”
```

---

## 两个练习和 OpenClaw 的区别

### OpenClaw / computer-use agent 更像

```text
看到屏幕
-> 决定点哪里
-> 控制鼠标键盘
-> 操作软件界面
```

### 这次作业更像

```text
读文字 / 读文档
-> 调用程序工具
-> 处理结构化任务
-> 输出结果
```

所以重点不是 GUI 自动化，而是：

1. 工具设计
2. 工具调用
3. 文档输入处理
4. 结果落地

---

## 最后再压缩成两句话

### 练习一

```text
做一个会调用“日程工具”的 Agent
```

### 练习二

```text
改一个“研究报告生成 Agent”的流程
```

如果你把这两个练习这样理解，基本就不会跑偏。
