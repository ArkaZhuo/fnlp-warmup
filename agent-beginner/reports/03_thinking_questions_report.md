## 3. 分题答案（工程化）

### 3.1 MCP 与 Skills 常见用法
- MCP：把外部系统（知识库、数据库、企业工具）以统一协议接入 Agent。
- Skills：把高频任务沉淀成可复用“能力包”（输入、步骤、输出规范）。
- 实践要点：
  - 一个 Skill 聚焦一类任务；
  - 明确输入输出 schema；
  - 设计失败回退（provider fallback）。

### 3.2 提高 Agent 的显卡利用率
- 推理服务层：使用支持连续批处理的推理框架（如 vLLM/TGI）。
- 请求层：批处理与并发控制，减少小请求碎片化。
- 模型层：KV Cache、量化、张量并行、显存占用参数调优。
- 系统层：压测驱动调参（吞吐、P95 延迟、GPU 利用率联动优化）。

### 3.3 利用环境反馈训练更好的 Agent
- 先采集：用户输入、工具调用轨迹、结果质量、人工修订。
- 再评估：构建可量化指标（任务完成率、错误率、成本、时延）。
- 再优化：
  - SFT 学习高质量轨迹；
  - 偏好优化（DPO/RLHF 类）提升策略质量；
  - 将线上失败样本滚动回灌评测集。

## 4. 我的思考
1. 思考题不是纯理论题，核心是“能否落到可观测指标和迭代闭环”。
2. 真正限制 Agent 体验的通常不是模型本身，而是外部工具成功率和系统工程能力。
3. 若做课程延伸，建议优先做一个最小反馈闭环（采集 -> 打标 -> 小步优化），比追求一次性大改更有效。
