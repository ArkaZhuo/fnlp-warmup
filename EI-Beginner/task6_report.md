# Task6 Report

## 任务内容（对应 README）
基于强化学习的人形机器人运动控制。本任务在 PyBullet humanoid 上实现了一个简化但完整的流程，包括全身遥操作轨迹生成、模仿学习策略训练以及强化学习残差微调，并对比不同阶段的跟踪误差。

## 代码简要介绍
核心脚本为 `scripts/task6_humanoid_rl_imitation.py`。本任务并未直接复现大规模人形机器人系统，而是在当前环境下实现一个与 README 目标一致的最小闭环：先得到全身协调动作，再通过学习方法复现并改进这些动作。

第一步是构造遥操作参考轨迹。脚本在 PyBullet humanoid 模型上选取双臂和双腿中的关键关节，利用正弦相位函数生成周期性全身动作，从而模拟一个稳定、协调的“全身遥操作参考控制器”。该参考轨迹并非真实人体动作采集数据，但它在功能上对应了“给定一个高质量全身动作源”的作用。

第二步是采集模仿学习数据。环境在执行参考轨迹的同时，记录当前关节位置、速度以及相位信息，再将这些观测与对应的参考关节目标组合成监督学习样本。这样便形成了一个标准的 `(observation, action)` 数据集。

第三步是训练模仿学习策略。脚本使用多层感知机对参考控制器进行行为克隆，使模型能够根据当前观测直接预测目标关节动作。经过训练后，再将该策略单独部署到环境中，评估其是否能够独立跟踪原始参考轨迹。

第四步是引入强化学习残差微调。在模仿学习策略的基础上，额外训练一个离散残差策略，用于对原始动作做小幅修正。其本质思想是：不让强化学习从零开始控制整个人形系统，而是让强化学习只负责“在已有模仿策略基础上做细微校正”，从而降低学习难度并提高稳定性。

第五步是对比三种控制方式的跟踪误差，即参考遥操作、纯模仿学习和“模仿学习 + RL 残差”三者的 tracking MSE。通过这一对比，可以直接观察强化学习微调是否真的带来了改进。

## 结果展示
![Task6 RL Training Curve](report_images/task6_rl_training_curve.png)

上图展示了 RL 残差策略训练过程中，人形机器人跟踪误差的变化趋势。纵轴为 tracking MSE，数值越低表示关节轨迹越接近遥操作参考轨迹。图中同时标出了纯模仿学习基线、遥操作参考误差和最终 RL 评估误差。

（1）纯模仿学习误差高于遥操作参考。遥操作参考误差为 `0.02529`，而纯模仿学习误差为 `0.09088`。可能原因是模仿策略虽然能拟合参考动作，但在闭环执行中会产生累计误差，尤其在人形机器人这种多关节耦合系统中更明显。

（2）RL 残差训练曲线存在波动。训练快照中误差并不是单调下降，例如中后期仍出现反弹。可能原因是本任务使用的是离散 Q-learning 残差策略，评估还受到初始相位、探索退火和仿真动态的影响，因此训练过程会比监督学习更不稳定。

（3）最终 RL 策略相对纯模仿学习有所改善。最终 RL 评估误差为 `0.08569`，低于纯模仿学习的 `0.09088`，但仍明显高于遥操作参考。说明 RL 残差确实起到了一定修正作用，但当前实现还只是简化验证版本，距离高精度人形机器人控制仍有差距。

## 观察到的结果
- `teleop_tracking_mse = 0.02529`
- `imitation_tracking_mse = 0.09088`
- `rl_tracking_mse = 0.08569`
- `rl_improvement_vs_imitation = +0.00520`
- RL 训练趋势数据记录：`rl_training_curve`

从数值上看，RL 残差相较于纯模仿学习带来了正向改进，说明“先模仿、再强化”的两阶段路线在这一类控制任务上是合理的。不过，最终结果与参考控制器之间仍然存在明显差距，说明当前系统尚未达到高精度控制水平。

## 自己的思考
这一任务反映出一个非常实际的问题：人形机器人控制难度远高于机械臂抓取，因为它不仅是动作预测问题，更是一个高维、强耦合、连续闭环控制问题。即便参考轨迹已经给出，模仿学习也很容易在实际闭环执行中产生累计误差。

因此，我认为“模仿学习 + 强化学习残差”是一个较为合理的技术路径。模仿学习负责快速获得基本动作结构，强化学习负责补偿闭环误差与动态偏差。后续若要进一步提高该任务的真实性和研究价值，可以考虑接入更真实的人体运动轨迹、增加更多关节维度，并在奖励函数中加入稳定性、能耗或平衡约束。
