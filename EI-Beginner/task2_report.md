# Task2 Report

## 任务内容（对应 README）
本任务要求完成两个层面的内容：

1. 学习强化学习基础知识，并在 OpenAI Gym / Gymnasium 环境中选择若干任务，使用强化学习方法训练并测试成功率。
2. 在 PyBullet / MuJoCo 中训练机械臂物体抓取策略，并尝试在真机部署，体会 Sim2Real 过程。

结合当前环境，本任务已经完成了 Gymnasium 离散任务训练与 PyBullet 机械臂抓取策略训练。MuJoCo 与真机部署部分由于当前环境未配置 MuJoCo、ROS 和真实机械臂接口，因此未实际部署，但在报告中对其可扩展方向进行了说明。

## 子任务一：Gymnasium 强化学习任务
### 实现目标
这一部分对应 README 中“在 OpenAI Gym 环境中选择几个感兴趣的 tasks 使用强化学习方法训练并测试成功率”的要求。其目的在于先验证强化学习的基本流程，包括状态、动作、奖励、策略、价值更新和评估方法。

### 代码简要介绍
对应脚本为 `scripts/task2_gym_qlearning.py`。该脚本选择了两个经典离散强化学习环境：

- `FrozenLake-v1`
- `Taxi-v3`

这两个任务都具有离散状态空间和离散动作空间，因此适合使用表格型 Q-learning。整体训练流程如下：

第一步是初始化 Q 表。对于每个状态和动作，维护一个 Q 值，用于估计“在当前状态下执行某个动作后能够获得的长期回报”。

第二步是使用 epsilon-greedy 策略进行探索。在训练早期，智能体会以较高概率随机选择动作，以探索环境；随着训练进行，epsilon 逐渐减小，智能体更多地选择当前 Q 表中价值最高的动作。

第三步是根据 Bellman 更新公式更新 Q 表。每执行一步动作后，利用即时奖励和下一状态的最大 Q 值更新当前状态-动作对的价值估计。

第四步是独立评估训练后的策略。训练完成后，关闭随机探索，使用贪心策略进行多轮测试，并统计成功率和平均回报。

## 子任务二：PyBullet 机械臂强化学习抓取
### 实现目标
这一部分对应 README 中“在 PyBullet/MuJoCo 中训练机械臂的物体抓取策略”的要求。当前项目中完成的是 PyBullet 版本，即训练 Panda 机械臂完成对方块的抓取和抬升。

### 代码简要介绍
对应脚本为 `scripts/task2_pybullet_qlearning_pick.py`。与 Gym 环境不同，机械臂抓取原本是一个连续控制问题。如果直接让 Q-learning 在连续末端位置和连续关节空间中学习，训练难度会非常高。因此本任务采用动作原语与状态抽象的方式，将机械臂抓取任务转化为一个可学习的离散强化学习问题。

第一步是构建 PyBullet 抓取环境。脚本加载 Panda 机械臂、桌面和方块，并定义抓取任务的初始状态、最大步数和成功条件。成功条件是方块被抓取并被抬升到一定高度。

第二步是设计离散动作原语。动作不再是连续关节力矩或连续末端速度，而是被抽象为若干高层动作，例如对齐、下降、抓取、抬升和随机微调。这种设计降低了动作空间复杂度，使 Q-learning 能够在较小状态空间内收敛。

第三步是进行状态抽象。环境状态被表示为几个布尔量组合，例如是否已经与方块对齐、末端是否足够低、是否已经抓住方块、方块是否已被抬升。这样可以将原本高维连续状态压缩为有限离散状态。

第四步是设计奖励函数。为了避免稀疏奖励导致训练困难，脚本引入了阶段性奖励，包括对齐奖励、下降接近奖励、成功抓取奖励和成功抬升奖励。这样智能体在完成最终目标之前，也能够从中间阶段获得学习信号。

第五步是训练与评估。训练阶段使用 epsilon-greedy 探索和 Q-learning 更新；评估阶段关闭探索，使用贪心策略执行抓取，并统计成功率。

## 结果展示与综合分析
本任务的结果展示不再使用最终成功率柱状图。最终评估成功率只能说明训练结束后的策略表现，例如本次实验中 `FrozenLake-v1`、`Taxi-v3` 和 `PyBullet-PickLift` 的最终评估成功率均为 `1.0`，但这个数值无法说明策略是如何学出来的，也容易让不同环境被误认为是同一种 benchmark。因此，这里保留两张训练过程变化趋势图，并将 Gymnasium 基础强化学习任务与 PyBullet 机械臂抓取任务放在一起分析。

需要特别说明的是，这两个子任务的目标并不完全相同。Gymnasium 部分主要用于验证强化学习算法流程，包括探索、Q 表更新、策略评估和成功率统计；PyBullet 部分则进一步把强化学习放到机械臂抓取场景中，通过状态抽象和动作原语把连续机器人控制问题转化为可以用 Q-learning 学习的离散决策问题。也就是说，前者更偏向强化学习基础实验，后者更接近 README 中“在 PyBullet/MuJoCo 中训练机械臂抓取策略”的机器人任务。

### 图一：Gymnasium 子任务训练趋势
![Task2 Gym Training Curve](report_images/task2_gym_training_curve.png)

图一展示了 `FrozenLake-v1` 和 `Taxi-v3` 两个 Gymnasium 环境在训练过程中的窗口成功率变化，统计窗口为 500 个 episode。这里的“成功率”指一个窗口内智能体成功完成任务的比例，而不是监督学习中分类意义上的准确率。

从曲线可以看出，`FrozenLake-v1` 的学习过程较为平滑。训练早期智能体主要依赖随机探索，因此前 500 轮窗口成功率只有 `0.092`；随着 Q 表不断更新，智能体逐渐学会从起点到目标格子的有效路径，到 2000 轮时窗口成功率提升到 `0.752`，5000 轮附近达到 `0.988`，后期基本稳定在 `0.97` 以上。这说明该任务中策略不是一开始就固定成功，而是经历了较明显的探索、修正和收敛过程。

`Taxi-v3` 的曲线则上升更快。前 500 轮窗口成功率为 `0.29`，到 1000 轮已经提升到 `0.94`，1500 轮后基本稳定在 `1.0`。这表明在当前训练参数下，Q-learning 较快学习到了“接乘客、移动、送达目标位置”的动作序列。虽然 `Taxi-v3` 的训练总 episode 数为 `22000`，多于 `FrozenLake-v1` 的 `7000`，但从曲线看，其有效策略在较早阶段已经形成，后续训练主要起到稳定 Q 值和保持策略鲁棒性的作用。

### 图二：PyBullet 机械臂抓取训练趋势
![Task2 PyBullet Training Curve](report_images/task2_pybullet_training_curve.png)

图二展示了 `PyBullet-PickLift` 机械臂抓取任务的训练趋势，统计窗口为 50 个 episode。与 Gymnasium 环境相比，这一部分的任务对象已经变成 Panda 机械臂和桌面方块，目标是完成对齐、下降、闭合夹爪、抓取并抬升方块的过程。

从曲线可以观察到，机械臂任务在训练前期成功率较低，前 50 轮窗口成功率为 `0.1`，说明智能体最初大部分 episode 还不能正确组合抓取动作。随着训练进行，成功率逐步上升：100 轮达到 `0.44`，200 轮达到 `0.66`，250 轮达到 `0.82`，到 300 轮后稳定达到 `1.0`。这一变化说明，策略逐渐学会了动作原语之间的顺序关系，例如不能在未对齐、未下降时过早闭合夹爪，也不能在未抓住方块时直接执行抬升。

需要注意的是，这里的 PyBullet 机械臂抓取并不是端到端连续控制强化学习，而是“离散状态抽象 + 动作原语 + Q-learning”的入门实现。这样设计的目的是降低训练难度，使强化学习能够在有限训练轮数内看到清晰的收敛趋势。它能够证明强化学习可以用于机械臂抓取策略训练，但还不能等同于复杂真实场景下的视觉闭环抓取系统。

### 综合观察
把两张趋势图放在一起看，可以得到更准确的结论。Gymnasium 子任务说明 Q-learning 的基础流程是有效的：通过 epsilon-greedy 探索收集经验，再用 Bellman 更新逐步改善 Q 表，最终形成稳定策略。PyBullet 子任务则说明，在机器人抓取场景中，如果直接学习连续控制会比较困难，因此需要先进行合理的任务抽象，把复杂抓取过程拆成对齐、下降、抓取和抬升等高层动作。

三个环境的曲线均体现出从低成功率到高成功率的变化过程，但它们不能被简单横向比较。`FrozenLake-v1` 和 `Taxi-v3` 是标准离散 RL 环境，主要考察路径选择和任务流程决策；`PyBullet-PickLift` 虽然也被离散化处理，但任务语义已经变为机械臂操作，其成功依赖末端位置、夹爪状态和方块高度等物理过程。因此，更合理的理解是：第一个子任务验证算法基础，第二个子任务验证该算法思想在机械臂抓取仿真中的迁移。

- `FrozenLake-v1` 训练轮数：`7000`
- `FrozenLake-v1` 后期训练窗口成功率：`0.978`
- `FrozenLake-v1` 最终评估成功率：`1.0`
- `Taxi-v3` 训练轮数：`22000`
- `Taxi-v3` 后期训练窗口成功率：`1.0`
- `Taxi-v3` 最终评估成功率：`1.0`
- `PyBullet-PickLift` 训练轮数：`500`

- `PyBullet-PickLift` 最终评估成功率：`1.0`

从结果上看，Gymnasium 部分验证了 Q-learning 在离散任务上的有效性；PyBullet 部分则说明，在进行合理动作原语设计、状态抽象和奖励塑形后，强化学习可以用于机械臂抓取策略训练。相比只展示最终 `1.0` 的成功率，训练趋势图更能说明本任务确实经历了学习过程，也能更清楚地看出不同任务的收敛速度和训练难度差异。

## 关于 MuJoCo、真机部署与 Sim2Real
任务要求中提到“在 PyBullet/MuJoCo 中训练机械臂抓取策略，并尝试在真机部署，体会 Sim2Real 过程”。本次工作中，PyBullet 仿真训练已经完成；MuJoCo 和真机部署尚未实际完成，原因是当前环境未安装 MuJoCo，也未接入实验室机械臂、ROS 控制接口和真机安全控制系统。

从方法上看，当前 PyBullet 版本可以作为 Sim2Real 的前置阶段。若后续迁移到 MuJoCo 或真机，需要重点处理以下问题：

- 仿真模型与真实机械臂的动力学差异
- 关节限位、速度限制和执行器延迟
- 相机或传感器观测误差
- 抓取接触与摩擦建模差异
- 真机安全边界与异常处理

因此，本任务已经完成了“仿真中训练机械臂抓取策略”的核心部分，但真机部署仍属于后续扩展内容。

## 自己的思考
本任务的一个重要体会是，强化学习能否成功，很大程度上取决于任务建模方式。在 Gym 这类离散环境中，Q-learning 的流程比较直接；但在机械臂抓取场景中，如果不进行动作原语设计和状态抽象，直接学习连续控制策略会困难得多。

从机器人角度看，当前实现更接近“高层动作决策强化学习”，而非完整的端到端低层控制。它的优点是训练稳定、结果可解释，适合入门理解强化学习在机器人抓取中的作用；局限是策略对当前任务结构依赖较强，泛化到复杂物体、视觉输入或真实机械臂时仍需要更强的表示能力和 Sim2Real 处理。

后续如果继续扩展，我认为较合理的路线是：先在 PyBullet 或 MuJoCo 中引入更丰富的随机化，例如物体位置随机化、摩擦随机化和视觉噪声；再逐步从离散动作原语过渡到连续动作控制；最后结合 ROS 或真实机械臂 SDK 完成真机部署验证。
